# Github API
- axios 활용을 하여 사용자가 키워드를 입력하면 해당 키워드가 포함된 리포지토리를 검색해서 가지고 올겁니다.
- TypeScript 써볼겁니다.
```tsx
import axios from 'axios';

type MyDataType = {
  id : number;
  name: string;
}

axios.get<MyDataType>(apiUrl)
  .then(response => console.log(response.data))
  .catch(error => console.log(error));
```
- 예상 데이터 타입에 없는 일부 field에 접근하려고 하면 개발 단계 초기에 오류가 발생합니다.
  해당 시점에서 TS는 JS로 컴파일되기 때문에 저희가 작성했던 Type에 대한 정보가 제거됩니다. 그래서 TS는 runtime 작동에 직접적인 영향을 끼치지 않습니다(빨간 줄은 떴는데 브라우저에서 빈값으로 나왔었습니다).
  REST API가 예상과 다른 데이터를 반환하는 경우에 TS는 이를 런타임 오류로 포착하지 않습니다.

- 리포지토리를 검색하기 위한 github REST API의 URL : https://api.github.com/search/repositories?q={검색어}

4. json으로 들어가서 확인해보시면 items key 내에 배열이 있다는 것을 확인할 수 있습니다. 그리고 내부 element는 객체로 이루어져있고, 거기서 각각의 repository 정보를 확인할 수 있네요.
  - 개별 repository 항목에서 full_name을 가지고 와서 githubid/repository_name으로 표시할 수 있도록 하겠습니다.
  - html_url을 가지고 와서 해당 리포지토리로 이동할 수 있도록 하겠습니다.

5. 사용자 입력의 검색어를 이용하여 REST API를 호출하도록 만들어보겠습니다.
  - 참고 : 이번에는 지난 번과 달리 useEffect()를 통해 REST API를 호출하는 것이 불가는ㅇ합니다. 이는 컴포넌트가 최초 렌더링되는 떄는 사용자가 입력한 키워드를 집어넣을 수 없기 떄문일겁니다.
    - 그런데 억지로 default 값을 만들고 싶다면 지금 생각나는 방법은 최초에 keyword 상태에 initialValue로 'korit_12'같은 것으로 초기화해두시면 최초 렌더링 시에도 겨로가값이 나오기는 할겁니다.

6. input 창에 값을 입력했을 떄 그것이 keyword 상태에 저장되어야 함.
7. 버튼을 클릭했을 때 GET 요청이 가야합니다. keyword를 포함해서. 현재 저희는 input ckddml onChange이벤트에 대해서는 arrow function을 썼지만, button의 onClick 이벤트의 경우에는 axios.get(어쩌고url + keyword) 해야하고, then()/catch() 등 chaining methods가 요구되기 때문에 onClick = {여기에 전부 나머지 로직을 다 집어넣기}는 좀 비효율적으로 보입니다. 그래서 handleClick이라고 하는 별개의 함수를 정의한 다음에 onClick={handleClick}으로 처리
```tsx
import { useState } from 'react';
import axios from 'axios';
import './App.css'

type Repository = {
  id: number; // 고유값을 통해서 나중에 .map() 적용했을 떄 사용
  full_name: string;
  html_url: string;
};

function App() {
  const [keyword, setKeyword] = useState('');
  const [repodata, setRepodata] = useState<Repository[]>([]);

  const handleClick = () => {
    axios.get<{items: Repository[]}>(`https://api.github.com/search/repositories?q=${keyword}`)
      .then(response => setRepodata(response.data.items))
      .catch(error => console.log(error));
  }

  return (
    <>
      <input type="text" onChange={event => setKeyword(event.target.value)} value={keyword}/>
      <button onClick={handleClick}>검색</button>
      { repodata.length === 0 ? (
        <p>검색 결과가 없습니다.</p>
      ) :(
        <table>
          <tbody>
            {repodata.map(repo => (
              <tr key={repo.id}>
                <td>{repo.full_name}</td>
                <td>
                  <a href={repo.html_url}>{repo.full_name}의 리포지토리입니다</a>
                </td>
              </tr>
            ))}
          </tbody>
        </table>
      )
      }
    </>
  )
}

export default App
```

## React Query 란 ?
- 서버에서 클라이언트로 데이터를 가져오거나, 동기화 및 업데이트를 쉽게 만들어주고 캐싱 기능을 지원해주면서 비동기 로직을 보다 쉽게 다루게 해주는 리액트 라이브러리입니다.
- 참고 : 캐싱(caching)이란 ? 동일한 데이터를 반복해서 가져오는 것을 줄이기 위해 메모리에 저장해두는 행위

## React Query의 사용 이유 ?
- 서버에서 받아온 데이터는 언제까지고 신선(fresh)한 상태라고 보장할 수 없습니다. 시간이 흘러서 사용자가 상호작용하여 데이터가 변했을 수도 있고, 서버에서 변경돼서 새로운 정보를 받아와야 할 수도 있습니다. 클라이언트에서 해당 로직들을 일일이 구현하는 것은 부담될 수 있겠네요.
여기서 React Query는 최신의 데이터를 간편하게 유지하는 다양한 방법을 제공합니다. 그리고 React Hook처럼 사용이 가능합니다.

## 설치 방법
`npm install @tanstack/react-query@버전` : 저희는 4로 설치

## React Query 세팅 및 사용 방법
```jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'
import {QueryClient, QueryClientProvider} from '@tanstack/react-query';
import './index.css'

const queryClient = new QueryClient();

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <QueryClientProvider client={queryClient}>
      <App />
    </QueryClientProvider>
  </React.StrictMode>,
)
```
- 이상의 예시와 같이 컴포넌트 최상단에서 `new QueryClient()` 생성을 통해 queryClient 인스턴스를 생성하고 `<QueryClientProvider client={queryClient}>` 와 같이 QueryClientProvider 컴포넌트에 client prop으로 전달하여 세팅을 완료함.

- 이상에서 용어 모르시면 복습하셔야 합니다.

## useQuery
### 형식
```jsx
import {useQuery} from '@tanstack/react-query';

const {data, isLoading. error} = useQuery(queryKey, queryFn, options);
```
- useQuery의 return 값은 이 세 가지 말고도 여러 속상값이 있습니다. 자주 사용하는 것 세 개만 적어놨습니다.

- useQuery는 **비동기 통신을 통해 GET 요청으로 데이터를 불러오는데 사용** 됩니다.
  비동기 통신이 끝나면 `isLoading`은 true에서 false로 바뀌고 `data`에 요청한 데이터가 담기게 되며, `error`는 오류 발생 시 return 됨.

### useQuery에 들어가는 argument
1. `queryKey` : useQuery 마다 부여되는 고유 key. 단순 문자열을 할 수 있고 배열로도 가능합니다.
2. `queryFn` : 쿼리가 데이터를 요청하는 데 사용하는 함수. fetch나 axios를 사용. 그러면 결과값의 자료형은 Promise다. 그걸 fetch API를 사용했다면 response.json()을 통해서 json 형태로 바꿔줬고 axios 를 썼다면 내부적으로 알아서 json으로 바꿔줬습니다.
3. `options` : 생략가능한 argument로 옵션 추가 시에 사용됩니다. enable, retry 등이 존재합니다.

### 사용 예시 설명

2. main.jsx 아까 위에 쓴 것처럼 react-query 적용
```jsx
import { useQuery } from '@tanstack/react-query'
import { fetchPosts } from './api'
import './App.css'

function App() {
  const {isLoading, data: posts} = useQuery({
    queryKey : ["posts"],
    queryFn : fetchPosts
  });

  return (
    <>
      {
        isLoading 
          ? '로딩중...'
          : posts.map((post, index) => <div>{`${index} : ${post.title}`}</div>)
      }
    </>
  )
}

export default App
```

export const fetchPosts = async () => {

- gitapi2 프로젝트 생성 App.jsx 초기화, axios, react-query 설치
- 이상의 예시에서는 main.jsx에 QueryClient와 QueryClientProvider를 적용했었습니다.
그런데 하위 컴포넌트에 해당하는 App에도 적용 가능하다고 명시했기 때문에 gitapi2에서는 App.jsx에 QuertClient인스턴스를 생성해보겠습니다.

```jsx
// App.jsx
import {QueryClient, QueryClientProvider} from '@tanstack/react-query'
import Repositories from './Repositories';
import './App.css'

const queryClient = new QueryClient();

function App() {
  

  return (
    <>
      <QueryClientProvider client={queryClient}>
        <Repositories />
      </QueryClientProvider>
    </>
  )
}

export default App

// Repositories.jsx
import axios from "axios";
import { useQuery } from "@tanstack/react-query";

export default function Repositories() {
  const getRepositories = async () => {
    const response = await axios.get('https://api.github.com/search/repositories?q=korit_12');
    return response.data.items;
  }

  const {isLoading, isError, data} = useQuery({
    queryKey: ['repositories'],
    queryFn: getRepositories,
  });
  
  if(isLoading) return <p>로딩 중...</p>
  
  if(isError) return <p>오류가 발생했습니다.</p>
  else {
    return (
      <table>
        <tbody>
          {
            data.map(repo => 
              <tr key={repo.id}>
                <td>{repo.full_name}</td>
                <td>
                  <a href={repo.html_url}>{repo.full_name}의 repository</a>
                </td>
              </tr>
            )
          }
        </tbody>
      </table>
    );
  }
}
```
- 이상과 같이 작성했습니다. fetch 사용했던 github REST API와 차이점으로 일단 input 없이 받아올 때 고정시켰습니다. 그 이유는 keyword를 사용하게 되면 기본적으로 useState()가 요구되기 때문입니다.
- useQuery 함수의 형식 : 
`const {isLoading, isError, isSuccess} = useQuery({queryKey : ['repositories'], queryFn: getRepositories})`
- 내부 속성들 정리
  1. queryKey : 쿼리를 위한 고유한 키로 데이터의 캐싱과 다시 가져오기에 이용됨.
  2. queryFn : 데이터를 가져오는 함수이며 Promise를 return 해야 합니다.

- useQuery Hook이 return하는 query 객체에는 이상과 같은 중요 속성이 있습니다(근데 const {} 라고 작서한 것을 봤을 때 **객체구조분해**가 적용되었음을 알 수 있습니다).
  1. isLoading : 데이터를 아직 이용할 수 있음.
  2. isError : 쿼리가 오류로 종료됨
  3. isSuccess : 쿼리가 성공적으로 종료되었으며 쿼리 데이터를 이용할 수 있음.
- 저희가 data로 정의한 부분 있습니다 -> useQuery()의 결과값인 query 인스턴스에는 응답을 return한 데이터가 포함됩니다.

- 이상의 코드를 다 작성하고 났을 때, 저희는 repodata와 같은 배열 상태를 정의하지 않았습니다.
  즉, 리액트 쿼리가 데이터 관리와 캐싱을 처리하기 때문에 응답 데이터를 저장하기 위한 상태가 필요없다는 것을 알 수 있습니다.

- repodata를 console.log 찍었을 때를 생각해보면 거기에 전체 json이 다 들어가있고, 그 중에 일부 데이터(id, full_name, html_url)만 상태에 저장해서 return 할 때 브라우저에 풀어줬었습니다. 하지만 react query를 사용했을 경우는 repodata라는 배열에 있는 Repository 자료형에 정의된 id, full_name, html_url 이외의 다른 것들도 정의하지 않고도 전부 다 쓸 수 있다는 점에서 강점이 있겠네요.
